# 화성복지플랫폼 (사회적협동조합)

> 흩어진 복지 정보를 한 곳에서 — 화성시 우정읍·장안면을 거점으로 시작해
> 전국 단위 복지 플랫폼으로 나아가는 지역 복지 통합·시각화 웹사이트.

경기도 화성시 **우정읍·장안면**의 인구·복지 현황·필요도를 통합하여
시각화하고, 복지 서비스와 자원을 분야별로 정리해 필요한 사람에게 연결합니다.

## ▶ 실행 방법

> ⚠️ **중요:** HTML 파일을 더블클릭해 `file://` 로 열면 **API(`/api/*`) 호출이
> 브라우저 보안정책에 막혀 동작하지 않습니다.** API를 받으려면 반드시
> **서버로 실행**해 `http://localhost:8787` 로 접속해야 합니다.

### 방법 1 — 원클릭 (Windows)

`start.bat` 더블클릭 → 의존성 자동 설치 후 브라우저가 열립니다.

### 방법 2 — 명령어

```bash
npm install     # 최초 1회 (server 의존성까지 자동 설치)
npm start       # → http://localhost:8787
```

- 홈페이지: `http://localhost:8787`
- 전국 지도: `http://localhost:8787/map.html`
- API: `http://localhost:8787/api/population` · `/api/nationwide` · `/api/health`

이 서버 하나가 **홈페이지(정적파일) + API 프록시**를 함께 제공합니다.

### 방법 3 — 공개 URL로 배포 (API 신청용 도메인 확보)

KOSIS·공공데이터 활용신청 시 **서비스(홈페이지) URL** 이 필요할 수 있습니다.
아래 중 하나로 무료 배포하면 공개 https 주소가 생깁니다.

| 방식 | 파일 | 특징 |
|---|---|---|
| **Render** | `render.yaml` | GitHub 연결 → 자동 배포, Node 풀스택(홈페이지+API) |
| **Docker** | `Dockerfile` | 어디서나 `docker build && docker run` |

```bash
# Docker 예시
docker build -t hwaseong-welfare .
docker run -p 8787:8787 hwaseong-welfare
```

> 배포 후 발급된 주소(예: `https://내앱.onrender.com`)를 KOSIS/공공데이터
> 활용신청의 서비스 URL에 등록하고, 발급키를 환경변수로 넣으면 실시간 연동 완료.

### 🗺️ 전국 복지 지도 (`map.html`)

17개 시도의 복지시설·복지지표 분포를 **타일그리드 추로플레스(choropleth)** 로
시각화합니다. 시설 종류를 바꾸면 지도·순위·표가 함께 갱신되고,
`절대수 ↔ 인구 10만명당`(정규화)을 토글해 지역 규모를 보정할 수 있습니다.

모든 수치는 **공식 통계 실값**입니다:

| 지표 | 전국 합계 | 출처 | 기준 |
|---|---|---|---|
| 노인여가복지시설(경로당·복지관 등) | 69,005개소 | KOSIS/보건복지부 | 2023 |
| 어린이집 | 26,064개소 | 보육통계(KOSIS) | 2025 |
| 등록장애인 | 2,627,761명 | 보건복지부(KOSIS) | 2025 |
| 총인구 / 고령화율 | 약 5,118만 / 19.6% | 행정안전부 | 2025 / 2024 |

> 정규화 시 **전남·전북·경북**(고령·농촌)이 인구당 노인시설 밀도 최상위,
> **수도권**이 최하위로 나타나 지역 격차가 드러납니다.

## 🔌 data.go.kr OpenAPI 연동 (server/ 프록시)

브라우저에서 data.go.kr 를 직접 부르면 **CORS·인증키 노출** 문제가 있어,
`server/` 의 프록시가 API를 대신 호출·가공·캐싱해 안전한 JSON(`/api/*`)을 내려줍니다.

### 1) 실행 (키 없이도 즉시 동작 — 공식 정적값 반환)

```bash
cd server
npm install
cp .env.example .env      # (Windows: copy .env.example .env)
npm start
# → http://localhost:8787  (대시보드가 자동으로 /api/population 을 읽어 갱신)
```

- `GET /api/population` — 우정읍·장안면 인구·세대·연령 통합 JSON
- `GET /api/nationwide` — 전국 17개 시도 복지시설 통계(KOSIS 연동)
- `GET /api/health` — 각 엔드포인트 모드(static/live) 확인
- 프런트( `data.js`/`nationwide.js` )가 이 엔드포인트를 자동 호출하며,
  프록시가 없으면 번들된 공식 데이터로 조용히 폴백합니다.
- 전국 실시간 조회(KOSIS)는 아래 절차로 연결합니다.

### 🔗 KOSIS OpenAPI 실시간 연동 (전국 지도)

KOSIS는 통계표마다 항목코드(`itmId`)가 달라, **KOSIS가 만들어주는 완성 URL을
붙여넣는 방식**이 가장 안정적입니다.

1. [kosis.kr/openapi](https://kosis.kr/openapi) 회원가입 → **활용신청 → 인증키 발급**
2. 각 통계표에서 **[OpenAPI] → 'URL 생성'** (시도 분류 `objL1=ALL`, `format=json`)
   - 노인여가복지시설 `DT_1YL20961` · 어린이집 `DT_15407_NN003` · 등록장애인 `DT_1YL202003E`
3. 생성된 URL(인증키 포함)을 `server/.env` 에 붙여넣기:
   ```env
   KOSIS_SENIOR_URL=https://kosis.kr/openapi/Param/statisticsParameterData.do?method=getList&apiKey=...&orgId=101&tblId=DT_1YL20961&objL1=ALL&itmId=...&prdSe=Y&newEstPrdCnt=1&format=json&jsonVD=Y
   KOSIS_DAYCARE_URL=...
   KOSIS_DISABLED_URL=...
   ```
4. URL이 올바른지 즉시 확인:
   `http://localhost:8787/api/kosis-test?url=<URL>` → 파싱 샘플이 나오면 정상
5. `npm start` → 전국 지도가 자동으로 **live 모드**(KOSIS 실데이터)로 갱신됩니다.

> 프록시가 KOSIS 시도명 변형(서울특별시/강원특별자치도/전북특별자치도 등)을
> 자동 정규화하므로 지도에 바로 매핑됩니다. 키·URL 미설정 시 번들된 공식 통계 사용.

### 2) 실시간 전환 (인증키 연결)

1. [공공데이터포털](https://www.data.go.kr) 로그인 → 아래 API **활용신청**
   - **행정안전부 주민등록 인구 및 세대현황 / 읍면동 연령별 인구수**
   - (추정 항목 실값용) **국민기초생활보장 수급자 현황**, **등록장애인 현황**,
     **노인복지시설·지역아동센터·어린이집 현황** 등
2. 발급받은 **일반 인증키(Decoding)** 와 데이터셋 URL 을 `server/.env` 에 입력:
   ```env
   DATA_GO_KR_KEY=발급키
   POP_API_URL=https://api.odcloud.kr/api/<데이터셋>/v1/uddi:<UDDI>
   ```
3. `npm start` → 자동으로 **live 모드**로 전환됩니다.
   응답 필드명이 데이터셋마다 달라 `server/server.js` 의 `mapRegion()` 에서
   키 매핑만 맞춰주면 됩니다. (예: `총인구수`, `세대수`, `65세이상` …)

### 데이터 스키마 (`WELFARE_DATA`)

| 키 | 내용 | 출처 |
|---|---|---|
| `regions` | 지역별 인구·세대·성별·고령화율·유소년율·1인세대율 | 공식 |
| `ageStructure` | 3대 연령(유소년·생산연령·고령) 인구 | 공식 |
| `gender` | 남/여 인구 | 공식(2025.08) |
| `trend` | 연도별 총인구·고령화율 추이(`real[]`로 공식/추정 구분) | 공식+추정 |
| `welfare.categories` | 복지 대상 현황 | 추정 |
| `facilities` | 복지 인프라(시설) 개소 | 추정 |
| `needs` | 5개 영역 필요도 지수(0–100) | 조합 지수 |
| `services` | 통합 복지 서비스 디렉터리 | — |

## 🎨 디자인·시각화 원칙

- **색상:** 데이터 시각화 검증 팔레트(카테고리 고정 순서, 색맹 안전) 적용.

  (이중 축 미사용).
- **접근성:** 범례 상시 표기 + 데이터 표 병행 + 상태색은 라벨·배지와 함께 사용,
  라이트/다크 모드 모두 지원.
